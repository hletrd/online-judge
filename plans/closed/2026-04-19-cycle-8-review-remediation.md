### S1 — [MEDIUM] ~~Fix `recordRateLimitFailure` inconsistent `consecutiveBlocks` exponent~~ — NOT A BUG
- **From:** AGG-1 (F1)
- **File:** `src/lib/security/rate-limit.ts:206`
- **Resolution:** On closer analysis, the code is consistent. The three call sites use different increment ordering but all effectively pass the original (pre-increment) `consecutiveBlocks`:
  - `consumeRateLimitAttemptMulti`/`recordRateLimitFailureMulti`: increment first (`consecutiveBlocks += 1`), then pass `consecutiveBlocks - 1` (effectively original value)
  - `recordRateLimitFailure`: pass `consecutiveBlocks` (original value), then increment (`consecutiveBlocks += 1`)
  - Both compute `calculateBlockDuration(original_consecutiveBlocks, blockMs)`. Test at `rate-limit.test.ts:148` confirms correct behavior. Initial fix was reverted in commit 3113722a.
- **Status:** NOT A BUG — reverted
