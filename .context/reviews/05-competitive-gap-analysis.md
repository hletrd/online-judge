## JudgeKit만의 독자적 강점

1. **124개 언어 지원** — BOJ(~60)의 2배. Docker 기반이라 추가도 쉽다. 아희, 혀엉, 엄준식 등 한국어 이색 언어 포함.
2. **현대적 기술 스택** — Next.js 16 + Rust. BOJ보다 유지보수성과 성능이 우수.
3. **Docker 샌드박스** — 언어 추가가 컨테이너 하나 추가로 끝남. ARM64도 지원.
4. **대회 리플레이 FLIP 애니메이션** — BOJ에 없고 Codeforces보다 시각적으로 우수.
5. **문제별 상세 피드백 설정** — BOJ보다 상세한 에러 피드백. RE 종류 분류, WA 케이스 인덱스.
6. **유사 문제 추천** — 태그 교집합 기반. BOJ에 없는 기능.
7. **교육 기능 통합** — 과제, 시험, 대회가 한 플랫폼에서. BOJ는 대회/연습만.
8. **안티치트(코드 유사도)** — 내장. BOJ는 외부 도구 필요.
9. **난이도 범위 필터 내장** — BOJ는 solved.ac 확장 필요.
10. **Quick Submit** — 데스크탑 Dialog + 모바� 바텀 Sheet. BOJ보다 모바일 경험이 우수.

## 결론: 실제 갭 요약 (v3 — 사실 기반)

### JudgeKit이 앞서는 영역 (v2보다 훨씬 많음)
- 언어 지원 수 (124개, 압도적)
- 현대적 UI/UX (다크모드, 반응형, Quick Submit Sheet)
- 교육 기능 통합 (과제+시험+대회)
- Docker 기반 확장성 + ARM64
- 파워 유저 단축키 (Ctrl+Enter, F11, Vim, N/P)
- 유사 문제 추천
- 가상 대회
- 클랜(Q&A) 시스템
- 대회 리플레이 (FLIP 애니메이션)
- 맞은 사람 코드 (3가지 정렬)
- 상세 에러 피드백 (RE 분류, WA 케이스 인덱스)
- 난이도 범위 필터 내장
- 문제집 내장
- 통합 검색

### JudgeKit이 뒤처지는 영역 (P0 — 기능 자체가 없음)
- **Special Judge** — 없으면 대회 출제가 제한됨. 최우선 해결 필요.
- **서브태스크 / 그룹 채점** — IOI 모드 필수.

### JudgeKit이 뒤처지는 영역 (P1 — 있어야 완전한 대회 플랫폼)
- **인터랙티브 문제** — 파이프 기반 IPC 필요.
- **팀 대회** — ICPC 타겟이면 필수.

### JudgeKit이 뒤처지는 영역 (P2 — 있으면 매력도 UP)
- **레이팅 시스템** — 참가자 동기부여 핵심.

### v2가 과소평가한 영역 (거의 모든 것)
v2가 "없다"고 한 기능의 **90% 이상이 실제로 구현되어 있었다.** 인라인 제출, 클랜, 맞은 사람 코드, 템플릿, 난이도 필터, 문제집, 통합 검색, 상세 피드백, 대회 공지, 리플레이 — 모두 있다.
