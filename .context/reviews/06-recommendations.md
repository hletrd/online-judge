# 06. 개선 제안 우선순위 (v3 — 정정판)

> **v3 노트**: v2가 Quick Submit, 템플릿 코드, 클랜, 맞은 사람 코드, 난이도 필터, 대회 공지, 통합 검색, 문제집, 상세 피드백, 리플레이를 "최우선 구현 필요"로 제안했으나, **모두 이미 구현되어 있음**이 확인됨. 이 제안 목록은 실제로 누락된 기능만 우선순위대로 정렬한다.

---

## P0 — 없으면 경쟁 프로그래밍 대회 출제가 제한됨

### 1. Special Judge (Checker) 시스템
- **이유**: 정답이 하나가 아닌 문제(최적해, constructive, 다중 정답) 출제 불가. 경쟁 프로그래밍 대회의 30% 이상이 special judge를 요구한다.
- **현재 상태**: 미구현. Rust 워커는 exact match / floating-point 비교만 지원.
- **구현 방안**:
  - 출제자가 C++ checker 프로그램을 업로드
  - `judge(input, user_output, expected_output) → AC/WA` 인터페이스
  - BOJ의 `spj` 스펙, Codeforces의 `check.cpp` 참고
- **예상 난이도**: 중간 (Rust 워커에 checker 실행 로직 추가)

### 2. 서브태스크 / 그룹 채점
- **이유**: IOI 모드를 지원한다면 서브태스크는 필수. "조건 A만족 → 30점, B까지 → 70점, C까지 → 100점" 형식.
- **현재 상태**: 미구현. 테스트케이스에 그룹 ID/점수 구조 없음.
- **구현 방안**:
  - 테스트케이스에 `groupId`와 `score` 필드 추가
  - 그룹 내 전부 AC 시 그룹 점수 획득
  - IOI 랭킹에 서브태스크 점수 시각화
- **예상 난이도**: 중간

## P1 — 있어야 완전한 대회 플랫폼

### 3. 인터랙티브 문제 지원
- **이유**: ICPC, IOI, Codeforces에서 빈출. 출제자의 인터랙터와 참가자 프로그램의 통신.
- **현재 상태**: 미구현. Docker 실행 환경에서 참가자 프로그램만 실행됨.
- **구현 방안**: Docker 컨테이너 내에서 파이프 기반 IPC. 인터랙터 프로세스 + 참가자 프로세스 동시 실행.
- **예상 난이도**: 높음

### 4. 팀 대회 지원
- **이유**: ICPC는 3명이 한 팀으로 참가하는 대회. 공동 제출, 팀 랭킹, 팀 관리.
- **현재 상태**: 미구현. DB 스키마에 팀 엔티티 없음.
- **구현 방안**: 팀 엔티티 추가, 팀원 관리 UI, 팀별 제출, 팀 랭킹.
- **예상 난이도**: 높음

## P2 — 있으면 플랫폼의 매력도가 크게 올라감

### 5. 레이팅 시스템
- **이유**: 경쟁 프로그래밍의 핵심 동기부여. Codeforces의 레이팅, solved.ac의 티어.
- **구현 방안**: 대회 성적 기반 ELO/Glicko 레이팅. 티어 색상 (브론즈/실버/골드/플래티넘/다이아). 레이팅 변화 그래프.
- **예상 난이도**: 높음

### 6. 제출 내역 대시보드 강화
- **이유**: 내 모든 제출을 한 곳에서 관리. 상태/날짜/문제별 필터.
- **현재 상태**: 제출 내역 페이지는 있으나 필터가 제한적.
- **구현 방안**: `/submissions` 페이지. 테이블 + 필터(AC/WA/TLE/MLE/RE/CE) + 날짜 범위 + 문제 검색 + 페이지네이션.
- **예상 난이도**: 중간

### 7. 대회 아카이브 & 통계
- **이유**: 과거 대회의 문제 세트, 제출, 순위를 모아보는 아카이브. 대회 후 분석.
- **구현 방안**: 대회 목록 페이지, 문제별 정답률, 참가자 분포.
- **예상 난이도**: 중간

## P3 — 있으면 디테일이 살아난다

### 8. 사용자 커스텀 템플릿 저장
- 기본 템플릿은 17개 언어로 제공되지만, 사용자가 자신만의 템플릿을 저장하는 기능은 없다.

### 9. 코드 공유 URL
- 내 코드를 URL로 공유. Pastebin 스타일. 현재는 "맞은 사람 코드"만 공개.

### 10. 입출력 구조화 강화
- 구조화된 파서가 있으나 마크다운 헤딩 기반. 문제 스키마에 `inputFormat`, `outputFormat`, `samples` 필드를 추가하면 더 견고해짐.

### 11. 영문 기술 용어 유지
- 한국어 UI에서도 AC, WA, TLE, DP, BFS, MST 등은 영문 그대로 표시.

### 12. OG 이미지 자동 생성
- 문제 공유 시 미리보기 카드 이미지. 문제 제목, 난이도, 정답률 포함.

### 13. 의미적 코드 유사도 분석
- 변수명 변경, 코드 재구조화를 탐지하는 고급 안티치트. MOSS 수준.

### 14. 문제 공개 제어
- 대회에서 문제 순차 공개, 조건부 공개. IOI 스타일.

### 15. 대회 통계 시각화
- 대회 종료 후 문제별 정답률, 참가자 분포, 난이도 분포.
