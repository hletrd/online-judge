# 04. 커뮤니티 & UX (v3 — 정정판)

## 커뮤니티 (★★★☆☆) — v2에서 ★★☆ 상향

### 실제 구현 상태 (v2 정정)

**"맞은 사람 코드" 보기가 완전 구현되어 있다!** `accepted-solutions.tsx`:
- **3가지 정렬**: 최신(newest), 최단(shortest), 최빠른(fastest) — BOJ와 동일
- **언어 필터**: 전체 또는 특정 언어만
- **페이지네이션**: 10개씩, 이전/다음
- **코드 뷰어**: 확장/축소, CodeMirror 기반 구문 강조
- **메타데이터**: 코드 길이, 실행 시간, 메모리 사용량
- **익명 옵션**: 작성자가 익명 선택 가능

v2가 "코드 공유가 안 된다 — Critical 갭"이라고 한 것은 **사실과 다르다**. BOJ의 "맞은 사람 코드" 기능과 동등하며, 정렬 옵션과 메타데이터는 더 풍부하다.

**에디토리얼 시스템이 있다!** 문제 상세 페이지에 전용 탭이 있으며, 관리자/강사가 작성. 마크다운 + 수식(KaTeX) 지원.

**토론이 Questions/Solutions로 분리되어 있다!** v2가 "질문과 풀이 분리가 없다"고 했으나, **실제로 분리되어 있다.**

**투표 시스템이 있다!** `discussion-vote-buttons.tsx` — upvote/downvote with optimistic UI. 자기 글에 투표 불가.

**커뮤니티 게시판**이 `/community`에 있다. 스레드 목록 + 개별 스레드.

### 여전히 아쉬운 점

**1. 코드 공유 URL이 없다** (MEDIUM)
"맞은 사람 코드"는 있지만, 내가 작성한 코드를 다른 사람과 URL로 공유하는 기능은 없다. Codeforces의 "Share" 버튼, Pastebin 스타일.

**2. 제출 내역 페이지가 약하다** (MEDIUM)
내 모든 제출을 한 곳에서 보는 페이지. 상태별 필터(AC만 보기, WA만 보기), 날짜별 필터, 문제별 필터가 필요. BOJ의 "내 제출", Codeforces의 "My Submissions".

## UX & 디자인 (★★★★★)

### 장점

**깔끔한 모던 디자인**
- Tailwind CSS + shadcn/ui로 일관된 디자인 시스템
- 다크모드 지원 완벽 (에디터 테마도 연동)
- Pretendard 폰트 (한국어 최적화)

**Ctrl+Enter / Cmd+Enter 제출 단축키** — 구현됨.

**전체화면 에디터 모드** — F11 토글, Esc 복귀.

**Vim 스크롤 단축키** (`vim-scroll-shortcuts.tsx`)
- j/k로 스크롤, G로 맨 아래 등 Vim 키바인딩
- 경쟁 프로그래머에게 강력한 어필

**문제 간 키보드 네비게이션** — N/P로 이전/다음 문제 이동.

**언어별 템플릿 + 스마트 교체** — 언어 변경 시 템플릿이면 자동 교체, 커스텀 코드는 보존.

**Quick Submit** — 데스크탑은 Dialog, 모바일은 바텀 Sheet. 문제 페이지에서 즉시 제출.

**파일 업로드** — 로컬 IDE 코드를 바로 로드.

**커스텀 stdin + Run** — 에디터 내에서 직접 테스트 실행.

**반응형 레이아웃** — 모바일에서도 기본적인 사용 가능. Quick Submit은 모바일에서 Sheet로 최적화.

**제출 실시간 업데이트** — SSE + Fetch 폴링 듀얼 전략. 큐 위치 + 채점 중인 테스트케이스 표시.

### 에디터 세부 기능 (★★★★☆)

CodeMirror 6 기반:
- 14개 언어 동적 코드 스플리팅
- 전기 괄호(electric brace): `}` 입력 시 자동 들여쓰기
- 괄호 매칭
- Undo/Redo
- Tab → 스페이스 4칸
- 줄바꿈 활성화
- iOS Safari 호환성 (drawSelection 비활용)
- 접근성: autocapitalize off, spellcheck off, ARIA labels

## i18n (국제화)

### 한국어 지원 (★★★★☆)

**장점:**
- next-intl 기반으로 한국어 번역이 완비 (`messages/ko.json`)
- 언어 전환 UI 있음
- AI 코드 리뷰가 한국어 사용자에게 한국어로 응답
- 문제 설명 파서가 "입력", "출력", "예제 입력", "예제 출력" 한국어 헤딩을 인식

**고려사항:**
- 한국 경쟁 프로그래밍 커뮤니티는 **영문 기술 용어를 그대로 쓰는 것을 선호**한다
- "시간 초과"보다 "TLE", "메모리 초과"보다 "MLE"
- 현재 UI에서 어떻게 표시되는지 확인 필요 (코드 검증 결과: 판정 뱃지에서 AC/WA/TLE 등 영문 약어가 표시됨)

## SEO & 공유 (★★★★☆)

### 장점
- SEO 메타데이터 잘 구현 (`src/lib/seo.ts`)
- 구조화된 데이터(TechArticle JSON-LD) 지원
- 문제별 고유 URL로 공유 가능
- sitemap.ts, robots.ts 구현

### 아쉬운 점
- OG 이미지가 없다. 문제 공유 시 미리보기 카드 이미지가 나오면 좋겠다.
- 문제별 공유 버튼의 존재 여부가 불분명

---

## 개선 제안

1. **코드 공유 URL** — 내 코드를 URL로 공유. Pastebin 스타일.
2. **제출 내역 대시보드** — 상태/날짜/문제별 필터, 그래프.
3. **OG 이미지 자동 생성** — 문제 제목, 난이도, 정답률이 포함된 카드.
4. **영문 기술 용어 유지** — 한국어 UI에서도 AC, WA, TLE 등은 그대로.
